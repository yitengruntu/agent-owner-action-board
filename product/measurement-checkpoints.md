# Measurement Checkpoints

The product is not waiting on SEO. It needs visible distribution to a narrow local-agent audience.

## Valid Exposure Window

A channel counts only if the post, comment, or listing remains visible to normal logged-out or non-owner readers.

Filtered, dead, removed, or moderator-hidden posts do not start a 24-hour or 72-hour decision window.

## 24-Hour Read

Collect:

- GitHub traffic: repository views, unique visitors, clones, and unique cloners.
- GitHub product signals: stars, forks, issues, comments, or connector requests.
- Public thread signals: comments, upvotes, saves, replies, or direct questions.
- Demo signal if available: GitHub Pages visits or any linked analytics.

Interpretation:

- Real comments asking about Codex, Claude Code, Cursor, cron, or a custom runner are stronger than passive views.
- Clones without comments are weak but worth one follow-up read.
- Removed or hidden posts are access failures, not demand failures.

## 72-Hour Read

Continue only if at least one is true:

- 5 or more direct technical interactions.
- 2 or more apparent install/clone attempts from visible distribution.
- 1 concrete connector request from a real user.
- A maintainer/community member asks for the tool in a specific workflow.

Park if:

- A visible post gets no technical replies.
- Repo traffic stays near-owner-only after a valid visible share.
- Interest is only generic praise with no workflow detail.
